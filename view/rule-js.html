JavaScriptコーディングルール

  • ECMAScript2016の仕様に準拠すること
  • UTF-8エンコーディングを使用し、改行コードはLFとすること
  • クラスごとにファイルを分割すること
    • かつ、export default class として定義すること
  • privateなフィールド及びメソッドは、アンダースコアから始まる名前とすること
  • 無名関数が必要な場合、function構文ではなくラムダを用いること
  • ローカル変数は原則的に const で宣言すること
  • 変数の初期化にはリテラルを用いること
    • new String('xxx') ではなく 'xxx'
    • new RegExp('xxx') ではなく /xxx/
    • new Array() ではなく []
    • new Object() ではなく {}
  • 可能な箇所では厳密比較演算子を用いること
    • == ではなく ===
  • リストのループには forEach() メソッドを用いること
